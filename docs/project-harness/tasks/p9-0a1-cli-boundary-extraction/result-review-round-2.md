## Remaining must-fix: contract bytes depend on inherited `HOME`

The implementation adds the caller's `HOME` back to `_allowed_env()`, uses
`Path.home()` as the help-normalization anchor, and compares the preserved legacy
`DEFAULT_DB_PATH` to `Path.home()/projects/coordinate/...`. This is correct only when
the process home happens to be `/Users/yinxin`.

Codex ran both:

```bash
HOME=/tmp/p9-fake-home \
  PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src \
  python3 -m unittest tests.test_cli_contract -v

HOME=/tmp/p9-fake-home \
  MULTI_AGENT_COORDINATOR_DB=/tmp/p9-poison.sqlite3 \
  PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src \
  python3 -m unittest tests.test_cli_contract -v
```

Each run failed:

1. the default-path assertion expected `/tmp/p9-fake-home/projects/...` while the
   behavior-preserving constant remained `/Users/yinxin/projects/...`;
2. contract generation failed to replace the source's `/Users/yinxin` help examples,
   so generated bytes differed from the reviewed `<HOME>` fixture.

The contract must be byte-identical across caller `HOME` values. Do not change runtime
`DEFAULT_DB_PATH` behavior in this structural package. Use an environment-independent
normalization anchor derived from the preserved default/source semantics rather than the
caller's home, and remove inherited `HOME` from the subprocess allowlist unless a fixed
value is explicitly necessary.

The exact default-path relationship must also be portable and privacy-safe. One viable
shape is to store and assert a SHA-256 of the exact default string in fixture metadata:
the exact value remains change-detecting without publishing it, while the parser default
still must equal `str(DEFAULT_DB_PATH)` under sanitized environment. An equivalently
strong design is acceptable.
