# Multi-Agent Collaboration

Multi-agent collaboration is one interaction surface of the Coordinate + MultiNexus
project execution layer. It is not the product's source of truth.

Start with the current documents:

1. [Product definition](project-harness/product-definition.md) — shared roles, invariants, delegation boundary, and fact ownership.
2. [MultiNexus scope](project-harness/scope.md) — execution-fabric responsibilities and non-goals.
3. [MultiNexus architecture](project-harness/architecture.md) — direct interaction, Coordinate-managed execution, agentd, adapters, and bridges.
4. [Coordinate operator skill](https://github.com/baisiqi6/multi-agent-coordinator/blob/main/skills/coordinate-operator/SKILL.md) — operational workflow for an AI acting as the current Operator.

Discord and KOOK remain useful visible collaboration surfaces. A message, handoff, or
agent reply becomes durable project evidence only when the appropriate harness,
Coordinate, Git, or forge authority records it.
